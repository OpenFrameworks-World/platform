# OpenFrameworks Canonical Capability Model

OpenFrameworks is capability-first.

Enterprises do not adopt frameworks for their own sake. They adopt frameworks, standards, protocols, methodologies, tools, and products to deliver capabilities.

## Capability Definition

A capability is a measurable ability required to achieve an outcome.

A capability may be enabled by:

- frameworks
- standards
- protocols
- methodologies
- reference architectures
- products
- services
- policies
- controls
- skills
- operating models

- id
- name
- domain
- description
- outcomes
- stakeholders
- enabling frameworks
- enabling standards

- enabling products
- controls

- metrics

## Capability-to-Framework Mapping

A framework may enable one or more capabilities.

A capability may require one or more frameworks.

Example:

```text
Capability: Identity Federation

Enabled by:
- OpenID Connect
- OAuth 2.0
- SAML
- SCIM

Implemented by:
- Keycloak
- Zitadel
- Authentik
- Okta
- Microsoft Entra ID
```

## Governance Rule

Framework recommendations must be grounded in capability needs.

A recommendation that does not declare the target capability is incomplete.
